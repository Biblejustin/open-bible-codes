## Why This Source Matters

This source family is useful because it differs from WRR/BCD-style statistical
claims. It emphasizes pictograms, symbolic images, overlapping matrices,
prophetic interpretation, and source-internal criteria for authenticity.

The introductory page also gives a compact ELS explanation:

- Torah codes are treated as a subset or near-synonym of Bible codes;
- ELS means equal-distance or equidistant letter sequences;
- hidden words can be read at positive skips such as 2, 3, or 4;
- backward/minus skips are explicitly allowed;
- spaces are removed for matrix display;
- a matrix slide is built by wrapping text at a fixed row width;
- changing row width changes which paths appear vertical or diagonal;
- John 3:16 in KJV is used as an English demonstration matrix;
- the source explicitly cautions that ordinary ELS words can arise by chance.

## Methodology Notes

This source reinforces several report requirements:

- matrix displays must declare row width and source text;
- diagonal or vertical appearance is a display choice, not an independent
  statistic;
- positive and negative skips must be recorded explicitly;
- English KJV examples should be analyzed separately from Hebrew/Greek primary
  hypothesis runs;
- "picture code" claims need a different metric from ordinary term hits;
- claims about overlapping pictograms require locked geometry before testing;
- prediction claims need archived timestamp evidence before evaluation;
- warnings against personal-name searches and horoscope-style use should be
  preserved as interpretive guardrails.

## Authenticity Criteria To Preserve

The source's own criteria are not standard statistics, but they are useful
claim metadata:

- numerical and symbolic coherence;
- acrostic or word-divider behavior;
- logical connection to other pictograms;
- textual variants modifying or recreating rather than simply destroying a
  code;
- direct Christological focus;
- prediction fulfillment.

If tested, these should be split into separate machine-checkable fields instead
of treated as one vague claim.

## Reproduction Requirements

Before elevating any Bible-codes.org row:

1. archive the source page and publication/update date;
2. lock exact corpus edition and normalization;
3. lock row width and matrix origin;
4. record every hidden letter path and surface text span;
5. define the image/pictogram scoring rule before looking at output;
6. separate ELS, acrostic, Atbash, numerics, prophecy, and pictogram claims;
7. include KJV and non-Bible controls where English examples are used;
8. apply family-level correction across all tested matrices, terms, row widths,
   and image interpretations.

## Cautions

- The source often rejects strict scientific framing, so its claims need extra
  pre-registration before empirical testing.
- The site contains devotional and interpretive claims; this project should
  preserve them as source claims, not adopt them as conclusions.
- Several examples are named by title without machine-readable letter paths.
- These additions expand the audit and term universe; they do not validate or
  invalidate the source claims by themselves.
